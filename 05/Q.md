# Маршрут до Пика Кода

Кодерун планирует маршрут на Пик Кода. На трёхмерной карте размещены N контрольных точек, каждая с координатами `xi, yi, zi`.

## Определение стоимости перехода

Стоимость перехода между двумя точками `i` и `j` определяется как минимум из модулей разностей их координат по осям x, y и z:

```
cost[i][j] = min(|xi - xj|, |yi - yj|, |zi - zj|)
```

## Задача

Нужно соединить все N контрольных точек ровно `N - 1` переходами так, чтобы из любой точки можно было добраться до любой другой (возможно, через промежуточные точки), и при этом суммарная стоимость была минимальной.

## Формат ввода

В первой строке вводится одно целое число N (1 ≤ N ≤ 100 000) — количество контрольных точек.

В следующих N строках вводятся по три целых числа: `xi, yi, zi` — координаты соответствующей контрольной точки (-10^9 ≤ xi, yi, zi ≤ 10^9).

## Формат вывода

Выведите одно целое число — минимальную суммарную стоимость, необходимую для соединения всех N контрольных точек с помощью `N - 1` переходов.

## Ограничения

- Ограничение времени: 3 секунды
- Ограничение памяти: 512 МБ

## Примеры

### Пример 1

**Ввод:**

```
2
2 6 11
8 9 3
```

**Вывод:**

```
3
```

**Пояснение:** Минимальная разность по координате y: |6 - 9| = 3. Достаточно сделать один переход для соединения двух контрольных точек.

### Пример 2

**Ввод:**

```
3
1 1 1
-5 -5 -5
-10 -10 -10
```

**Вывод:**

```
11
```

**Пояснение:** Разности по всем координатам равны. Можно сделать переход от первой контрольной точки ко второй, и от второй к третьей, в результате все точки будут соединены.

### Пример 3

**Ввод:**

```
5
-2 -2 -6
10 -16 -16
18 -5 18
13 -16 -16
9 -5 -2
```

**Вывод:**

```
4
```
