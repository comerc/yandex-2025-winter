20. Солнечный Процессор

Кодерун входит в главный зал замка Лорда Нуля. В центре возвышается замёрзший Солнечный Процессор, окутанный непроницаемым защитным алгоритмом. Но Кодерун догадывается: чтобы разрушить защиту, нужно вычислить истинный $xormex$ — значение, которое делает алгоритм уязвимым.

У Кодеруна есть последовательность из $n$ целых неотрицательных чисел. Обозначим за $xormex$ последовательности наибольшее значение величины $mex(a_1 \oplus x, a_2 \oplus x, \dots, a_n \oplus x)$ для некоторого целого неотрицательного $x$. Ваша задача — найти $xormex$ данной последовательности.

Обработайте $q$ запросов. В $i$-м запросе вам даны числа $j, v$ ($1 \le j \le n$) и вам нужно выполнить присвоение $a_j = v$. После каждого запроса выведите также $xormex$ последовательности.

### Формат ввода

В первой строке вам дано единственное число $T$ ($1 \le T \le 10^4$), обозначающее количество тестовых примеров.

В следующих строках следует описание тестовых примеров:

- В первой строке каждого тестового примера содержатся числа $n$ и $q$ ($1 \le n, q \le 2 \cdot 10^5$), обозначающее размер последовательности и количество запросов.
- В следующей строке тестового примера содержатся $n$ целых неотрицательных чисел $a_i$ ($0 \le a_i < 2^{30}$), задающие последовательность.
- Следующие $q$ строк задают запросы — по два целых числа $j, v$ ($1 \le j \le n, 0 \le v < 2^{30}$), обозначающие индекс изменяемого элемента и значение, которое ему нужно присвоить.

Гарантируется, что сумма $n$ по всем тестовым примерам не превосходит $2 \cdot 10^5$ и сумма $q$ по всем тестовым примерам не превосходит $2 \cdot 10^5$.

### Формат вывода

Для каждого тестового примера выведите $q + 1$ чисел — $xormex$ последовательности до запросов и после каждого запроса.

Как только алгоритм будет взломан, Солнечный Процессор заработает вновь. Да наступит цифровая оттепель! Но странно вот что... Где Лорд Нуль? И почему никто не помешал Кодеруну взломать защиту?

### Примечание

Напомним, что:

- Побитовая операция XOR двух целых неотрицательных чисел $u$ и $v$ (обозначается $u \oplus v$) — это число, в двоичной записи которого в каждом бите стоит 1, если соответствующие биты $u$ и $v$ различны, и 0 в противном случае.
- MEX ($minimum$ $excluded$ $value$) для набора целых неотрицательных чисел — это наименьшее целое неотрицательное число, не входящее в данный набор. Более формально:
  $mex(S) = \min\{t \ge 0 | t \notin S\}$.
