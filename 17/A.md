# Решение задачи 17: Защитное поле

## Краткое описание

Дано число `n`. Нужно найти такие два положительных целых числа `a` и `d` (длиной не более `2*10^5` цифр), что для арифметической прогрессии `x_k = a + k*d` (где `k = 0..n-1`), последовательность сумм цифр `S(x_k)` также образует арифметическую прогрессию.

**Ограничения:**

- `1 <= n <= 10^4`
- Время: 1 секунда
- Память: 256 МБ

## Математическая модель

### Условие задачи

Требуется найти `a, d >= 1` такие, что:
`S(a + k*d) = S_0 + k*D'` для некоторого `D'`.

### Идея решения

Попробуем найти такое `d` и `a`, чтобы сумма цифр `S(a + k*d)` была константой (то есть арифметической прогрессией с разностью 0).

Рассмотрим числа, состоящие только из девяток.
Пусть `M` — натуральное число.
Пусть `a = 10^M - 1` (число из `M` девяток).
Пусть `d = 10^M - 1` (тоже число из `M` девяток).

Тогда `k`-й член прогрессии:
`x_k = a + k*d = (10^M - 1) + k*(10^M - 1) = (k+1)*(10^M - 1)`.

Пусть `m = k + 1`. Тогда `x_k = m * (10^M - 1)`.
Так как `k` изменяется от `0` до `n-1`, то `m` изменяется от `1` до `n`.

Преобразуем выражение для `x_k`:
`x_k = m * 10^M - m`.
Так как `m >= 1`, можем записать `m = (m-1) + 1`.
`x_k = (m-1) * 10^M + (10^M - m)`.

Здесь `(m-1) * 10^M` — это число `m-1`, сдвинутое на `M` позиций влево (дописано `M` нулей).
`10^M - m` — это дополнение `m` до `10^M`.

Если `m < 10^M` (то есть количество цифр в `m` не превышает `M`), то эти две части не "перекрываются" при сложении (или перекрываются тривиально, если `m` мало).
На самом деле, `10^M - m` — это число, которое в сумме с `m` дает `10^M`.
Если рассмотреть десятичную запись:
Пусть `M` достаточно велико, чтобы `m` помещалось в `M` разрядов.
Тогда `10^M - m` имеет вид:
`99...9` (всего `M` девяток) минус `m`.
Это стандартное дополнение до девяток. Сумма цифр числа `10^M - m` (если `m < 10^M`) равна `9*M - S(m-1)`? Нет.
Давайте проверим на примере. `M=2` (99), `m=2`.
`100 - 2 = 98`. `S(98) = 17`.
`S(m-1) = S(1) = 1`.
`Sum = 18 = 9*2`.

Утверждение: **Если `X < 10^M`, то `S(X) + S(10^M - 1 - X) = 9*M`.**
Доказательство: `10^M - 1` состоит из `M` девяток. Вычитание `X` из `99...9` происходит поразрядно без заёмов. Если `X = d_k...d_0`, то результат `(9-d_k)...(9-d_0)`. Сумма цифр результата: `sum(9 - d_i) = 9*M - S(X)`.
То есть `S(10^M - 1 - X) = 9*M - S(X)`.

Вернемся к нашему числу:
`x_k = m * (10^M - 1) = m * 10^M - m`.
Мы хотим вычислить `S(x_k)`.
Запишем `x_k = (m-1)*10^M + (10^M - m)`.
Заметим, что `10^M - m = (10^M - 1) - (m - 1)`.
Обозначим `X = m - 1`. Тогда вторая часть это `(10^M - 1) - X`.
Первая часть `X * 10^M` — это просто цифры `X` и `M` нулей.
Вторая часть `(10^M - 1) - X` — это `M` цифр (возможно с ведущими нулями, которые не влияют на сумму, но по факту число положительное и меньше `10^M`).
Так как `X < 10^M` (это условие мы должны обеспечить выбором `M`), то число `x_k` получается конкатенацией цифр числа `X` и цифр числа `(10^M - 1) - X`.
Следовательно, `S(x_k) = S(X) + S((10^M - 1) - X)`.
Используя утверждение выше: `S((10^M - 1) - X) = 9*M - S(X)`.
Получаем: `S(x_k) = S(X) + (9*M - S(X)) = 9*M`.

**Вывод:** Сумма цифр `S(x_k)` постоянна и равна `9*M` для всех `k`, при условии, что `m = k+1 < 10^M`.
Максимальное значение `k` равно `n-1`, значит максимальное `m` равно `n`.
Условие корректности: `n < 10^M`.

### Алгоритм

1. Находим минимальное `M` такое, что `10^M > n` (или `10^M > n-1`, но для надежности возьмем строго больше `n`).
   Так как `n <= 10^4`, то `M` будет небольшим (максимум 5).
   Однако, в задаче сказано `a` и `d` до `2*10^5` цифр.
   Мы можем просто взять `M` достаточно большим, например, таким, что `10^M > n`.
2. Выбираем `a = 10^M - 1` (строка из `M` девяток).
3. Выбираем `d = 10^M - 1` (строка из `M` девяток).
4. Выводим `a` и `d`.

## Сложность

### Временная сложность

Поиск `M`: `O(log n)`.
Генерация строк: `O(M)`.
Так как `M` пропорционально количеству цифр в `n` (очень мало) или мы можем сразу взять `M` чуть больше, сложность линейна от длины ответа.
Для вывода используется буферизованный ввод-вывод.

### Пространственная сложность

Хранение строк `a` и `d`: `O(M)`. `M` порядка 5-6 для `n=10000`. Память ничтожна.

## Корректность

Приведенное математическое доказательство гарантирует, что `S(a + k*d)` будет константой (арифметической прогрессией с разностью 0) для всех допустимых `k`.

## Пример

`n = 10`.
`10^1 = 10` (не строго больше `n=10`? условие `m < 10^M` означает `n < 10^M`).
Берем `M=2`. `10^2 = 100 > 10`.
`a = 99`, `d = 99`.
Прогрессия:
k=0: 99 (S=18)
k=1: 198 (S=1+9+8=18)
...
k=9: 990 (S=9+9+0=18)
Все суммы равны 18.
